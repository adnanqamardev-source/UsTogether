## Brief overview
This set of guidelines is project-specific and covers the preferred project structure, coding conventions, and communication style for the UsTogether project.

## Communication style
- The user prefers a concise communication style, with a focus on providing clear and direct instructions.
- The user values simplicity and clarity in code and documentation.

## Development workflow
- The user prefers a iterative development workflow, with a focus on testing and verifying results at each stage.
- The user values collaboration and open communication throughout the development process.

## Coding best practices
- The user prefers to follow standard coding conventions and best practices for the chosen programming language.
- The user values code readability and maintainability.

## Project context
- The user prefers to keep the project structure organized and easy to navigate.
- The user values consistency in naming conventions and coding style throughout the project.
